# Herald 1.8.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Herald 1.8.0 as a single release containing reliability/recovery fixes (B1–B13) and the local-first iPad Notes workspace with native PencilKit editing.

**Architecture:** iOS app (SwiftUI + UIKit) ↔ FastAPI relay (Python) ↔ Python connector ↔ Hermes API server. Notes feature adds a dedicated local-first data layer with PencilKit ink as the canonical artifact, on-device handwriting recognition, and Hermes-powered enrichment via dedicated relay endpoints.

**Tech Stack:** Swift 6.2, SwiftUI, UIKit, PencilKit, Core Data/File-based persistence, FastAPI, PostgreSQL/SQLite, Python 3.x, XcodeGen, WidgetKit

## Global Constraints

- **Single release target:** Herald 1.8.0. No separate 1.7.6, 1.7.7, or other 1.7.x releases.
- **Version source of truth:** `project.yml` (XcodeGen). Never edit `project.pbxproj` directly.
- **Deployment target:** iOS 18.0 minimum.
- **Swift concurrency:** Strict concurrency mode `complete`.
- **Device families:** iPhone (1) and iPad (2).
- **Notes is local-first in 1.8.0:** No cross-device ink sync. iPad editing, iPhone read-only recognized/enriched view.
- **Apple Notes refinement is NOT available:** No public API for Smart Script/refinement. Herald provides its own recognition only.
- **Relay topology:** `https://your-relay.example.com/v1` → Caddy on INTERNAL_HOST → public relay container. NOT the `.118:8010` stack.
- **APNs environment:** Development tokens use sandbox; TestFlight/App Store tokens use production. Per-registration routing required.
- **No `git push origin master`** until approved 1.8.0 release after all gates pass.
- **No retroactive 1.7.x tags.** The unversioned commit `3804d82` is part of the 1.8.0 baseline.
- **All work on scoped branches:** `codex/` or maintainer-approved feature branches.

func parseTerminal(_ envelope: [String: Any]) -> TerminalResult {
    guard let messageField = envelope["message"] else {
        return TerminalResult(text: nil, reasoning: nil)
    }
    
    // Handle serialized message object
    if let messageDict = messageField as? [String: Any],
       let content = messageDict["content"] as? String {
        let reasoning = envelope["reasoning"] as? String
        return TerminalResult(text: content, reasoning: reasoning)
    }
    
    // Handle legacy string form
    if let messageString = messageField as? String {
        let reasoning = envelope["reasoning"] as? String
        return TerminalResult(text: messageString, reasoning: reasoning)
    }
    
    return TerminalResult(text: nil, reasoning: nil)
}
